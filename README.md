# High Passes

High Passes is a serverless travel enquiry platform built on AWS services and designed to run entirely on **MiniStack** for local development. The application demonstrates a modern event-driven architecture using API Gateway, Lambda, DynamoDB, Cognito, S3, SQS, SES, and Terraform/OpenTofu.

---

# Architecture

```
                        React Frontend
                               │
                               │
                         Amazon Cognito
                               │
                               ▼
                         API Gateway
                               │
               ┌───────────────┴───────────────┐
               │                               │
               ▼                               ▼
        Writer Lambda                  Reader Lambda
               │                               │
               │                               ▼
               │                         DynamoDB
               │
      ┌────────┼────────┐
      │        │        │
      ▼        ▼        ▼
 DynamoDB      S3      SES
      │         │
      │         ▼
      │       SQS
      │         │
      │         ▼
      └──► Processor Lambda
```

# Configure the Frontend

Open:

```
src/config.js
```

Update it with the values generated by MiniStack.

```javascript
const config = {

    region: "us-east-1",

    cognitoEndpoint: "http://localhost:4566",

    userPoolId: "<USER_POOL_ID>",

    clientId: "<CLIENT_ID>",

    apiGateway:
        "http://localhost:4566/restapis/<API_GATEWAY_ID>/dev/_user_request_/items"

};

export default config;

export const API_BASE_URL =
    "http://localhost:4566/restapis/<API_GATEWAY_ID>/dev/_user_request_";
```

# Find the API Gateway ID

Run

```bash
awslocal apigateway get-rest-apis
```

Example

```json
{
  "items": [
    {
      "id": "8fa35a50",
      "name": "items-api"
    }
  ]
}
```

Use the value of **id**.

Example

```javascript
apiGateway:
"http://localhost:4566/restapis/8fa35a50/dev/_user_request_/items"

export const API_BASE_URL =
"http://localhost:4566/restapis/8fa35a50/dev/_user_request_";
```

# Find the Cognito User Pool ID

Run

```bash
awslocal cognito-idp list-user-pools --max-results 60
```

Example

```json
{
  "UserPools": [

      "Id": "us-east-1_JMPuvVPrc",
      "Name": "highpasses-users"
    }
  ]
```

```javascript
userPoolId: "us-east-1_JMPuvVPrc"
```

# Lambda Functions

## Writer Lambda

Responsible for:

- Validating user input
- Rate limiting requests
- Preventing duplicate submissions
- Generating enquiry IDs
- Creating geohashes
- Saving enquiries to DynamoDB
- Sending confirmation emails using SES
- Returning API responses

---

## Reader Lambda

Responsible for:

- Reading enquiries from DynamoDB
- Returning enquiry data as JSON
- Supporting authenticated API access through Cognito

---

# DynamoDB

The application uses two tables.

## enquiries

Stores all customer enquiries.

Example record

```json
{
  "id": "...",
  "name": "...",
  "email": "...",
  "tour": "...",
  "status": "pending",
  "emailSent": true
}
```

## rate_limits

Used for

- Rate limiting
- Duplicate submission detection

# S3

Bucket

```
www.highpasses.com
```

# SQS

Queue

```
processor-queue
```
